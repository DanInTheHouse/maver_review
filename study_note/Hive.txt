# Hive

하둡에서 정형화된 상태의 데이터 처리를 위한 인프라로, 

일반적으로 대용량의 정형 및 반정형 데이터를 다루는 데 사용

## 역할

1. 대용량 데이터 저장 및 관리
    
    Hadoop Distributed File System (HDFS)와 같은 분산 파일 시스템을 기반으로 하여 데이터를 안정적으로 저장하고 관리
    
2. SQL 기반 쿼리 언어 제공
    
    기존의 데이터베이스 시스템과의 호환성을 제공하면서도, Hadoop의 분산 처리 능력을 활용할 수 있도록 함.
    
3. 비정형 데이터 구조화
    
    ETL 프로세스 지원으로 비정형 데이터를 구조화된 형태로 매핑하는 ETL(추출, 변환, 로드) 작업에 적합
    
    → 사용자는 대규모 데이터셋에서 의미 있는 정보를 추출하고 분석할 수 있음.
    
4. 분산 데이터 처리
    
    Hadoop 생태계의 일부로서, 분산 환경에서 대규모 데이터셋을 효율적으로 처리
    
    → 여러 노드에 데이터를 분산 저장하고 병렬 처리를 통해 빠른 쿼리 실행
    

### → 주로 대규모 데이터 처리와 분석을 수행하는 환경에서 사용되며, Hadoop 클러스터와 통합하여 데이터 처리 및 분석의 효율성을 높일 수 있다.
